## 一、论文概览

| 论文 | 核心贡献 | 关键技术 |
|------|----------|----------|
| **P01 - Disambiguate First** | 两阶段消歧：先解释后解析 | LLM 解释生成 + 填充模型 |

## 二、歧义分类对比

### AMBROSIA 的语言学分类

| 类型 | 定义 | 示例 |
|------|------|------|
| **Scope** | 量词指代范围不明确 | "each gym offer" → 集体/分布解释 |
| **Attachment** | 修饰语附着位置不明确 | "writers and editors on work-for-hire" |
| **Vagueness** | 实体集合不明确 | "Who issued CD Special?" |

### AmbiSQL 的实现分类

| 类型 | 定义 | 示例 |
|------|------|------|
| **DB 相关** | Schema/值/关键词歧义 | "最老用户" → 年龄/注册日期 |
| **LLM 相关** | 知识来源/推理上下文/冲突 | "女性员工" → 查列/语义分析 |

### 我们的三层框架

| 层面 | 定义 | 对应 AMBROSIA | 对应 AmbiSQL |
|------|------|---------------|--------------|
| **用户层** | 个人语言特征 | - | 推理上下文不足 |
| **业务层** | 领域业务知识 | Scope, Attachment | 知识来源、知识冲突 |
| **数据层** | Schema 歧义 | Vagueness | Schema 引用、值引用 |

**关键差异**：
- AMBROSIA：语言学视角（句法结构）
- AmbiSQL：系统实现视角（DB vs LLM）
- 我们：信息流视角（输入→语义→实现）

## 三、消歧方法对比

### 方法分类

| 方法 | P01 | P03 | P04 |
|------|-----|-----|-----|
| **消歧时机** | SQL 生成前 | SQL 生成前 | SQL 生成过程中 |
| **交互方式** | 非交互 | 多选题交互 | 澄清问题交互 |
| **输出形式** | 自然语言解释 + SQL | 重写问题 + SQL | SQL |
| **用户参与度** | 无 | 高（每歧义点确认） | 中（仅关键歧义点） |

### 技术路线

```
                    ┌─────────────────────────────────────┐
                    │         P01: 两阶段消歧              │
                    │   问题 → 解释生成 → SQL 解析         │
                    │   (非交互，自动生成所有解释)         │
                    └─────────────────────────────────────┘
                                    ↓
                    ┌─────────────────────────────────────┐
                    │         P03: 交互式消歧              │
                    │   问题 → 歧义检测 → 多选题确认       │
                    │            → 问题重写 → SQL         │
                    │   (交互，用户确认每个歧义点)         │
                    └─────────────────────────────────────┘
                                    ↓
                    ┌─────────────────────────────────────┐
                    │         P04: 最优问题选择            │
                    │   问题 → SQL候选 → EIG选择           │
                    │            → 澄清问题 → 更新候选     │
                    │   (交互，仅关键歧义点)               │
                    └─────────────────────────────────────┘
```

## 五、对我们研究的启示

### 1. 歧义分类

- AMBROSIA 的分类偏语言学，适用于学术研究
- **我们的三层框架更实用**：对应不同的解决策略
  - 用户层 → 用户画像学习
  - 业务层 → 知识图谱构建
  - 数据层 → 元数据增强

### 2. 消歧方法

| 启示 | 说明 |
|------|------|
| **P01** | 两阶段思路可借鉴：先理解意图，再生成 SQL |
| **P03** | 多选题交互设计值得参考，但交互频次需优化 |
| **P04** | EIG 策略可解决"最小交互"问题 |

### 3. 研究空白

| 空白 | 现有方法 | 我们的机会 |
|------|----------|-----------|
| **用户层消歧** | 仅 P03 简单提及角色差异 | 用户画像 + 历史学习 |
| **业务知识注入** | P12-P14 但非专门针对消歧 | 业务知识图谱 |
| **三层协同消歧** | 各方法独立解决某一层面 | 统一框架 |
