## 一、总体框架设计

### 1.1 框架概述

**框架名称**：**TriCo-DAG** (Three-Layer Collaborative Disambiguation for Data Agents)

**核心思想**：
> 将用户层画像、业务层知识、数据层元数据纳入统一决策框架，通过**证据融合**和**动态权重**机制实现协同消歧，阻断歧义传播。

**设计原则**：
| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **分层解耦** | 三层独立处理，降低复杂度 | 各层独立推理模块 |
| **证据融合** | 多层证据加权决策 | 动态权重融合算法 |
| **歧义阻断** | 高置信度层可阻断传播 | 置信度阈值机制 |
| **增量学习** | 每次交互更新知识 | 用户画像 + 知识图谱双更新 |

### 1.3 核心算法流程

```python
def TriCoDAG_disambiguate(query, user_id, context):
    """
    三层协同消歧主算法
    
    参数:
        query: 用户自然语言查询
        user_id: 用户标识
        context: 对话上下文
    
    返回:
        (sql, confidence, explanation) 或 (clarification_question, candidates)
    """
    
    # ========== Step 1: 歧义检测 ==========
    ambiguities = AmbiguityDetector.detect(query, context)
    # ambiguities = [(type, description, prior_prob), ...]
    
    if len(ambiguities) == 0:
        # 无歧义，直接生成 SQL
        sql = generate_sql(query, user_id, context)
        return (sql, 1.0, "无歧义，直接执行")
    
    # ========== Step 2: 三层并行推理 ==========
    # 2.1 用户层推理
    user_evidence, c_u = UserLayerInference.infer(
        query, user_id, get_user_profile(user_id)
    )
    
    # 2.2 业务层推理
    business_evidence, c_b = BusinessLayerInference.match(
        query, get_knowledge_graph()
    )
    
    # 2.3 数据层推理
    schema_evidence, c_d = DataLayerInference.link(
        query, get_schema_metadata()
    )
    
    # ========== Step 3: 协同决策 ==========
    # 3.1 置信度校准 (温度缩放)
    c_u_cal = calibrate(c_u, temperature=1.5)
    c_b_cal = calibrate(c_b, temperature=1.2)
    c_d_cal = calibrate(c_d, temperature=1.0)
    
    # 3.2 动态权重计算
    w_u, w_b, w_d = compute_dynamic_weights(
        query_type=query.type,
        user_confidence=c_u_cal,
        business_confidence=c_b_cal,
        schema_confidence=c_d_cal,
        context=context
    )
    
    # 3.3 证据融合 (加权乘积)
    joint_posterior = fuse_evidence(
        user_evidence, business_evidence, schema_evidence,
        weights=(w_u, w_b, w_d)
    )
    
    # 3.4 决策
    best_intent = argmax(joint_posterior)
    max_confidence = joint_posterior[best_intent]
    
    if max_confidence > THRESHOLD_EXECUTE:
        # 高置信度：直接执行
        sql = generate_sql(best_intent, user_id, context)
        # 隐式更新用户画像
        update_user_profile(user_id, query, best_intent, feedback='implicit')
        return (sql, max_confidence, build_explanation(best_intent))
    
    else:
        # 低置信度：交互确认
        # 基于 EIG 选择最优问题
        best_question = select_best_question_EIG(
            joint_posterior, ambiguities
        )
        candidates = get_top_k_intents(joint_posterior, k=3)
        return (best_question, candidates)
```

### 2.3 画像更新算法

```python
def update_user_profile(user_id, query, resolved_intent, feedback_type):
    """
    用户画像增量更新算法
    
    参数:
        user_id: 用户 ID
        query: 原始查询
        resolved_intent: 消歧后的意图
        feedback_type: 'implicit' | 'explicit'
    """
    profile = get_user_profile(user_id)
    
    if feedback_type == 'explicit':
        # 显式反馈：高学习率
        lr = 0.3
        
        # 更新概念偏好
        for concept, interpretation in resolved_intent.concepts.items():
            if concept in profile.concept_preferences:
                # 提升选中解释的权重
                old_weight = get_weight(profile.concept_preferences[concept], interpretation)
                new_weight = old_weight + lr * (1 - old_weight)
                update_weight(profile.concept_preferences[concept], interpretation, new_weight)
            else:
                # 新概念：初始化
                profile.concept_preferences[concept] = [(interpretation, 0.5)]
        
        # 归一化
        normalize_preferences(profile.concept_preferences)
        
    else:  # implicit
        # 隐式反馈：低学习率
        lr = 0.05
        
        # 基于查询模式更新表达习惯
        profile.expression_style['口语'] += lr * (1 if is_colloquial(query) else -0.5)
        profile.expression_style['简称'] += lr * (1 if has_abbreviations(query) else -0.5)
        
        # 基于查询内容更新角色视角
        if query.contains('个人') or query.contains('我的'):
            profile.role_perspective[get_main_concept(query)] = '个人视角'
    
    # 更新画像置信度
    profile.profile_confidence = min(1.0, profile.profile_confidence + 0.02)
    profile.last_update = datetime.now()
    profile.total_interactions += 1
    
    # 保存
    save_user_profile(profile)
```

### 4.2 用户耐心建模

**问题**：连续多轮确认会导致用户放弃。

**解决**：建模用户耐心，动态调整确认策略。

```python
@dataclass
class UserPatienceModel:
    """用户耐心模型"""
    
    # 静态因素
    user_data_literacy: float  # 数据素养
    historical_patience: float  # 历史耐心评分
    
    # 动态因素
    current_round: int  # 当前确认轮次
    accumulated_time: float  # 累计交互时间 (秒)
    query_complexity: float  # 查询复杂度
    recent_frustration_signals: List[float]  # 挫折信号 (重新查询/取消)
    
    def patience_score(self) -> float:
        """计算当前耐心评分 [0, 1]"""
        base = self.historical_patience
        
        # 轮次惩罚 (指数衰减)
        round_penalty = 0.3 ** (self.current_round - 1)
        
        # 时间惩罚
        time_penalty = exp(-self.accumulated_time / 300)  # 5 分钟半衰期
        
        # 挫折信号
        frustration_factor = 1.0 - np.mean(self.recent_frustration_signals)
        
        return base * round_penalty * time_penalty * frustration_factor
    
    def should_stop_confirming(self, expected_additional_rounds: int) -> bool:
        """判断是否应该停止确认"""
        if self.patience_score() < 0.3:
            return True
        
        # 预期总轮次超过阈值
        if self.current_round + expected_additional_rounds > 5:
            return True
        
        return False
```

### 4.3 最小交互策略算法

```python
def select_best_question_EIG(posterior, ambiguities, patience_model):
    """
    基于 EIG 和用户耐心的最优问题选择
    
    参数:
        posterior: P(意图│证据) 联合后验分布
        ambiguities: 检测到的歧义列表
        patience_model: 用户耐心模型
    
    返回:
        (question, expected_rounds)
    """
    # 1. 生成候选问题集
    candidate_questions = []
    for ambiguity in ambiguities:
        for template in get_templates_for_type(ambiguity.type):
            q = instantiate_template(template, ambiguity)
            eig = compute_EIG(q, posterior)
            candidate_questions.append((q, eig, ambiguity))
    
    # 2. 按 EIG 排序
    candidate_questions.sort(key=lambda x: x[1], reverse=True)
    
    # 3. 考虑用户耐心
    if patience_model.should_stop_confirming(expected_additional_rounds=2):
        # 耐心不足：选择单轮信息量最大的问题
        best_q = max(candidate_questions, key=lambda x: x[1])
        return best_q[0], 1
    
    # 4. 多轮策略：选择能最大化总信息增益的问题序列
    # 使用贪心近似
    selected_questions = []
    remaining_entropy = entropy(posterior)
    
    while remaining_entropy > THRESHOLD_UNCERTAINTY:
        # 选择当前 EIG 最大的问题
        best_q = candidate_questions.pop(0)
        selected_questions.append(best_q)
        
        # 更新剩余熵
        remaining_entropy -= best_q[1]
        
        # 检查耐心
        if patience_model.should_stop_confirming(
            expected_additional_rounds=len(selected_questions)
        ):
            break
    
    # 5. 返回第一个问题
    return selected_questions[0][0], len(selected_questions)
```

### 5.1 动态权重计算

**核心思想**：根据查询类型和上下文动态调整各层权重。

```python
def compute_dynamic_weights(query_type, user_confidence, business_confidence, 
                           schema_confidence, context):
    """
    动态权重计算
    
    返回:
        (w_u, w_b, w_d) 满足 w_u + w_b + w_d = 1
    """
    # 基础权重 (默认平等)
    base_weights = {'user': 0.33, 'business': 0.34, 'data': 0.33}
    
    # ========== 规则 1: 查询类型调整 ==========
    if query_type == '个人业绩查询':
        # 个人查询：用户层权重高
        base_weights['user'] += 0.2
        base_weights['business'] -= 0.1
        base_weights['data'] -= 0.1
    
    elif query_type == '业务指标查询':
        # 业务指标：业务层权重高
        base_weights['business'] += 0.2
        base_weights['user'] -= 0.1
        base_weights['data'] -= 0.1
    
    elif query_type == '技术探索查询':
        # 技术探索：数据层权重高
        base_weights['data'] += 0.2
        base_weights['user'] -= 0.1
        base_weights['business'] -= 0.1
    
    # ========== 规则 2: 置信度调整 ==========
    # 高置信度层获得更高权重
    confidences = {
        'user': user_confidence,
        'business': business_confidence,
        'data': schema_confidence
    }
    
    # Softmax 归一化置信度权重
    conf_weights = softmax([confidences[k] for k in ['user', 'business', 'data']])
    
    # ========== 规则 3: 融合 ==========
    # 基础权重 + 置信度权重的加权平均
    alpha = 0.6  # 基础权重占比
    
    final_weights = {}
    for key in ['user', 'business', 'data']:
        final_weights[key] = alpha * base_weights[key] + (1 - alpha) * conf_weights[key]
    
    # 归一化
    total = sum(final_weights.values())
    return (final_weights['user']/total, 
            final_weights['business']/total, 
            final_weights['data']/total)
```

### 5.2 证据融合算法

```python
def fuse_evidence(user_evidence, business_evidence, schema_evidence, weights):
    """
    证据融合：加权乘积法则
    
    参数:
        user_evidence: Dict[intent_id, probability]
        business_evidence: Dict[intent_id, probability]
        schema_evidence: Dict[intent_id, probability]
        weights: (w_u, w_b, w_d)
    
    返回:
        Dict[intent_id, joint_probability]
    """
    w_u, w_b, w_d = weights
    
    # 获取所有意图
    all_intents = set(user_evidence.keys()) | set(business_evidence.keys()) | set(schema_evidence.keys())
    
    joint_posterior = {}
    
    for intent in all_intents:
        # 获取各层证据 (无证据时为均匀分布)
        p_u = user_evidence.get(intent, 1.0 / len(all_intents))
        p_b = business_evidence.get(intent, 1.0 / len(all_intents))
        p_d = schema_evidence.get(intent, 1.0 / len(all_intents))
        
        # 加权几何平均
        joint = (p_u ** w_u) * (p_b ** w_b) * (p_d ** w_d)
        
        joint_posterior[intent] = joint
    
    # 归一化
    total = sum(joint_posterior.values())
    return {k: v/total for k, v in joint_posterior.items()}
```

### 5.3 歧义阻断机制

```python
def check_ambiguity_blocking(confidences, thresholds):
    """
    歧义阻断检查
    
    如果某一层置信度超过阈值，可以阻断歧义传播。
    
    参数:
        confidences: (c_u, c_b, c_d)
        thresholds: (θ_u, θ_b, θ_d)
    
    返回:
        (blocked, blocking_layer, blocking_confidence)
    """
    c_u, c_b, c_d = confidences
    θ_u, θ_b, θ_d = thresholds
    
    # 检查各层是否达到阻断阈值
    if c_u >= θ_u:
        return (True, 'user', c_u)
    if c_b >= θ_b:
        return (True, 'business', c_b)
    if c_d >= θ_d:
        return (True, 'data', c_d)
    
    return (False, None, 0.0)
```

## 六、算法复杂度分析

### 6.1 时间复杂度

| 组件 | 复杂度 | 说明 |
|------|--------|------|
| 歧义检测 | O(n) | n = 查询长度 |
| 用户层推理 | O(1) | 画像查找 O(1)，偏好匹配 O(k) |
| 业务层推理 | O(m × d) | m = 概念数，d = 图谱深度 |
| 数据层推理 | O(s × l) | s = Schema 元素数，l = 链接深度 |
| 证据融合 | O(i) | i = 候选意图数 |
| EIG 计算 | O(q × a × i) | q = 问题数，a = 回答数，i = 意图数 |

**总体**：O(n + m×d + s×l + q×a×i)，实际中 i, q, a 较小，主要瓶颈在图谱检索和 Schema Linking。

### 6.2 空间复杂度

| 组件 | 空间 | 说明 |
|------|------|------|
| 用户画像 | O(u × f) | u = 用户数，f = 特征数 |
| 知识图谱 | O(c + r) | c = 概念数，r = 规则数 |
| Schema 元数据 | O(t × col) | t = 表数，col = 列数 |
| 对话上下文 | O(h × q) | h = 历史轮次，q = 平均查询长度 |
