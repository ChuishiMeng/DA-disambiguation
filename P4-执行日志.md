### 2.2 框架设计 (22:35-22:45)

#### 2.2.1 三层协同消歧框架 (TriCo-DAG)

**设计决策**：
- 命名：TriCo-DAG (Three-Layer Collaborative Disambiguation for Data Agents)
- 架构：并行三层推理 + 协同决策引擎
- 核心机制：证据融合 + 动态权重 + 歧义阻断

**关键组件**：
```
用户查询 → 歧义检测器 → 三层并行推理 → 协同决策引擎 → 直接执行/交互确认
            │              │                    │
            │              ├─ 用户层画像推理    │
            │              ├─ 业务层知识匹配    │
            │              └─ 数据层 Schema 链接  │
            │                                   │
            └───────────────────────────────────┘
                     置信度阈值判断
```

#### 2.2.2 用户画像学习机制

**设计要点**：
- 双通道学习：冷启动（隐式）+ 迭代优化（显式）
- 画像表示：消歧专用结构（概念偏好、表达习惯、角色视角）
- 更新算法：显式反馈高学习率 (0.3)，隐式反馈低学习率 (0.05)
- 冷启动策略：角色先验 + 相似用户 + 主动询问

#### 2.2.3 业务知识图谱表示

**Schema 设计**：
- 节点类型：Concept（业务概念）、Rule（计算规则）、Dimension（维度）、Constraint（约束）
- 边类型：SYNONYM_OF、COMPUTED_BY、DEPENDS_ON、CONFLICTS_WITH、MAPPED_TO
- 版本管理：增量变更 + 影响分析 + 冲突检测
- 消歧优化：多跳检索 + 规则过滤 + 置信度计算

#### 2.2.4 交互式消歧策略

**核心算法**：
- EIG + 模板混合：结合理论最优性和自然度
- 用户耐心建模：轮次惩罚 + 时间惩罚 + 挫折信号
- 最小交互策略：贪心近似选择问题序列
- 决策树：置信度阈值 → 耐心检查 → 单轮/多轮确认

#### 2.2.5 三层协同决策算法

**动态权重计算**：
- 基础权重：查询类型调整（个人/业务/技术）
- 置信度调整：Softmax 归一化
- 融合方式：加权几何平均

**证据融合**：
- 算法：加权乘积法则
- 输入：各层后验分布
- 输出：联合后验分布

**歧义阻断**：
- 机制：某层高置信度可阻断传播
- 阈值：用户层 0.95，业务层 0.90，数据层 0.85

## 四、关键设计决策

### 4.1 为什么选择加权乘积融合？

**考虑方案**：
1. 加权平均：简单但忽略证据独立性
2. 加权乘积：假设条件独立，适合多层证据
3. D-S 证据理论：复杂，参数多

**决策**：加权乘积  
**理由**：
- 三层证据相对独立（用户/业务/数据）
- 计算简单，易于实现
- 可解释性强

### 4.2 为什么动态权重而非固定权重？

**场景分析**：
- 个人业绩查询：用户层更重要
- 业务指标查询：业务层更重要
- 技术探索查询：数据层更重要

**决策**：动态权重  
**理由**：
- 适应不同查询类型
- 利用上下文信息
- 提升消歧准确率

### 4.3 为什么 EIG + 模板混合？

**纯 EIG 问题**：
- 计算开销大
- 生成问题可能不自然

**纯模板问题**：
- 缺乏理论保证
- 无法适应新歧义类型

**决策**：混合策略  
**理由**：
- EIG 保证信息效率
- 模板保证自然度
- LLM 重写提升体验
