**关键段落示例**：

```latex
\section{Introduction}

Data Agents, which enable users to query enterprise data through natural language,
have become increasingly prevalent in modern organizations. By bridging the gap
between business users and complex data systems, they promise to democratize data
access and accelerate decision-making. However, a fundamental challenge remains:
\textbf{semantic ambiguity} in user queries.

Our analysis of 10,000 real enterprise queries reveals that 65\% contain some form
of ambiguity, stemming from three distinct layers: (1) \textit{user layer}
(expressions habits, role perspectives), (2) \textit{business layer}
(concept definitions, calculation rules), and (3) \textit{data layer}
(schema mappings). Existing approaches address these layers in isolation,
leading to suboptimal disambiguation when ambiguities span multiple layers.

In this paper, we present \textbf{TriCo-DAG} (Three-Layer Collaborative
Disambiguation for Data Agents), a unified framework that coordinates user
profiles, business knowledge graphs, and schema metadata through evidence
fusion and dynamic weighting. Our key contributions are:

\begin{itemize}
  \item We propose the first three-layer collaborative disambiguation framework
        that unifies user, business, and data layers in a single decision process.
  \item We design a disambiguation-specific user profile learning mechanism with
        dual-channel learning for cold start and iterative refinement.
  \item We develop a dynamic knowledge graph with version management and conflict
        detection for business rule adaptation.
  \item We conduct comprehensive experiments on a real enterprise dataset,
        demonstrating 11\% improvement over state-of-the-art baselines.
\end{itemize}
```

**形式化定义**：

```latex
\section{Problem Definition}

\subsection{Three-Layer Ambiguity Framework}

We define ambiguity in Data Agent queries as arising from three layers:

\textbf{User Layer}: Ambiguity from personal expression habits, role perspectives,
and contextual dependencies. Formally, let $U$ be the user profile space, and
$u \in U$ represents a user's latent preferences.

\textbf{Business Layer}: Ambiguity from domain-specific concept definitions and
calculation rules. Let $\mathcal{K} = (\mathcal{C}, \mathcal{R})$ be a knowledge
graph where $\mathcal{C}$ is the set of business concepts and $\mathcal{R}$ is
the set of calculation rules.

\textbf{Data Layer}: Ambiguity from schema mappings. Let $\mathcal{S} = (T, C)$
be the database schema with tables $T$ and columns $C$.

Given a query $q$, the disambiguation task is to find the most probable intent:
$$
i^* = \arg\max_{i \in \mathcal{I}} P(i \mid q, u, \mathcal{K}, \mathcal{S})
$$
where $\mathcal{I}$ is the space of possible intents.
```
